# Security Policy

## Supported Versions

We actively monitor and patch vulnerabilities for the following versions of our project:

| Version | Supported          |
| ------- | ------------------ |
| v2.x    | :white_check_mark: |
| v1.x    | :x:                |

## Reporting a Vulnerability

We take the security of our application seriously. If you discover any security vulnerabilities, please do **not** open a public GitHub issue. Instead, report them responsibly by following these steps:

1. **Email us directly:** Send a detailed report to [your-security-email@example.com] (e.g., security@edyfra.com).
2. **Include details:** Please provide a clear description of the vulnerability, steps to reproduce it, and any potential impact or proof of concept (PoC).

### Our Process

* **Acknowledgement:** We will acknowledge receipt of your report within 48 hours.
* **Evaluation:** Our team will investigate the issue and determine the severity.
* **Resolution:** If confirmed, we will work on a fix. We aim to resolve critical issues as quickly as possible.
* **Disclosure:** We will coordinate a public disclosure timeline with you once a patch is deployed.

Thank you for helping keep our platform and community safe!
